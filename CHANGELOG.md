# 更新日志

## v1.11.2 (2026-07-10)

### 修复
- 修复 session 超时后页面不会自动跳转登录的问题：前端新增会话心跳检测（每分钟验证一次）
- 修复浏览器从 bfcache 恢复标签页时 session 已过期但页面仍可操作的问题
- 修复自动保存模式在 session 过期时静默失败导致数据丢失的问题
- 所有 API 调用统一拦截 401 状态码，自动弹出提示并跳转登录页

## v1.11.1 (2026-07-04)

### 修复
- 修复生成邀请码时提示信息显示原始 HTML 标签的问题
- 邀请码生成后新增专属展示区块 + 一键复制按钮

## v1.11.0 (2026-07-04)

### 新增
- 部署模式：后台新增「部署模式」设置面板，一键在内网便捷模式和外网安全模式之间切换
  - 内网便捷模式：开放注册、无登录限速、密码最短4位
  - 外网安全模式：仅邀请注册、登录限速（5次失败锁定15分钟）、密码最短8位
- 邀请码系统：外网安全模式下管理员可生成/删除邀请码，用户注册须提供有效邀请码
- 登录暴力破解防护：启用后同一 IP 在规定次数内连续失败将临时锁定
- 自定义安全设置：支持单独调整注册模式（开放/邀请/关闭）、密码长度限制、登录限速参数

### 修复
- 修复 api.php 中 updated_at 和 deleted_at 仍使用 SQLite CURRENT_TIMESTAMP（UTC）的时区 Bug，统一改用 PHP date() 传入本地时间
